````md
# LogPilot 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称

LogPilot

### 1.2 产品定位

LogPilot 是一个 AI 驱动的代码仓库日志治理工具，通过分析代码中的日志使用情况，帮助开发者发现无效日志、缺失日志以及不符合规范的日志，并提供自动优化建议。

产品目标：

- 提升代码仓库日志质量
- 降低无效日志带来的维护和成本压力
- 提高线上问题定位效率
- 让日志成为代码运行状态的重要信息载体

### 1.3 产品形态

初期：

- 本地 CLI 工具
- 面向个人开发者和小型团队
- 支持本地代码仓库扫描分析

后续：

- IDE 插件
- CI/CD 集成
- 团队协作平台
- 云端日志治理平台

# 3. 产品目标

## 3.1 核心目标

通过 AI 分析代码上下文，自动识别：

- 无价值日志
- 缺失关键日志
- 日志规范问题
- 潜在敏感信息泄露


并提供：

- 优化建议
- 修改方案
- 自动代码修复能力


## 3.2 非目标

MVP 阶段暂不实现：

- 线上日志采集
- 日志存储
- 日志查询分析平台
- 完整 DevOps 平台
- 多团队权限管理
````

# 5. MVP 核心功能

## 5.1 代码仓库扫描

### 功能描述

扫描用户指定代码仓库，识别项目中的日志调用。

支持：

* Python
* Java
* JavaScript / TypeScript

后续扩展：

* Go
* Rust
* C++
* PHP

### 扫描内容

识别：

* logger.info
* logger.debug
* logger.warning
* logger.error
* console.log
* print
* System.out.println

### 输出信息

包括：

* 文件路径
* 代码位置
* 日志级别
* 日志内容
* 上下文代码

示例：

```
文件:
payment/service.py

位置:
Line 35

日志:
logger.info("start payment")
```

---

# 5.2 AI 日志质量分析

## 功能描述

结合代码上下文和业务逻辑，对日志价值进行分析。

## 分析类型

### 无效日志检测

识别：

* 无业务含义日志
* 重复日志
* 调试残留日志

示例：

代码：

```python
logger.info("enter function")
```

分析结果：

```
问题:
低价值日志

原因:
无法提供业务状态信息

建议:
删除
```

---

### 缺失日志检测

识别：

* 关键流程缺少状态日志
* 异常处理缺少错误日志
* 重要业务节点无记录

示例：

代码：

```python
try:
    payment.process()
except Exception:
    pass
```

分析结果：

```
问题:
异常被捕获但没有日志记录

建议:
增加 error 日志

推荐:

logger.error(
 "payment_failed",
 exc_info=True
)
```

---

### 日志规范检测

检测：

* 日志字段缺失
* 日志格式不统一
* 敏感信息打印

示例：

问题：

```python
logger.info(user.password)
```

分析：

```
风险:
敏感信息泄露

建议:
禁止输出密码字段
```


# 5.4 自动代码优化

## 功能描述

根据 AI 分析结果生成代码修改建议。

支持：

* 删除无效日志
* 修改日志内容
* 增加关键日志

示例：

修改前：

```python
logger.info("start")
```

修改后：

```python
logger.info(
    "payment_started",
    extra={
        "order_id": order.id
    }
)
```

输出：

* Diff 文件
* Patch 文件

---

# 5.5 用户确认修改

命令：

```bash
logpilot apply
```

执行：

* 应用 AI 修改
* 保留修改记录
* 支持回滚

# 6. 技术方案

## 6.1 系统架构

```
代码仓库

    |
    ↓

代码解析模块

(Tree-sitter / AST)

    |
    ↓

日志节点提取

    |
    ↓

AI分析模块

    |
    ↓

优化建议生成

    |
    ↓

代码Patch生成

```

---

## 6.2 核心模块

### Repository Scanner

职责：

* 遍历代码文件
* 识别语言类型
* 提取日志调用

### Log Analyzer

职责：

* 理解日志上下文
* 判断日志价值
* 发现缺失日志

### Rule Engine

职责：

* 执行固定规则检测
* 支持用户自定义规则

### Code Modifier

职责：

* 生成修改方案
* 输出代码Diff
